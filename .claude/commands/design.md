# 디자인만 수정 (내용 동결): $ARGUMENTS

**디자인 구역(레이아웃·클래스·테마)만** 수정한다. 내용의 말은 한 글자도 바꾸지 않는다.

## 동결 규칙 (반드시 준수)
- ✅ 수정 허용: `<!-- _class: ... -->` Marp 지시, `<div class="...">` HTML 래퍼, **클래스 마커 `{.box .accent}`·`::: ... :::`**, `themes/*.css`, `_header.md`의 theme/footer
- 🚫 절대 금지: 제목·불릿·문단·코드의 **글자(말)** 변경. **강조 마크(`**굵게**`·`*초록*`·`==형광펜==`)도 내용 작성자 소유 → 추가/제거/이동하지 않는다**(색은 테마가 입힘).
- 텍스트는 그대로 두고 감싸는 구조·스타일만 바꾼다. 다른 레이아웃으로 옮길 때도 **문구는 동일하게** 유지한다.

## 진행
1. 대상: deck-name + (번호 주면 그 파일 / 없으면 전체 또는 테마). `decks/<deck-name>/slides/` 또는 `themes/tech.css`.
2. 지시사항대로 레이아웃·색·간격·클래스만 조정. 필요하면 `patterns/PATTERNS.md` 레이아웃 적용. 내용 텍스트는 보존.
3. `node build.mjs <deck-name>` 빌드 확인 후, **무엇을 바꿨는지(내용은 안 건드렸음)** 보고.

> 글자(내용)를 바꾸려면 `/content`를 쓴다.
