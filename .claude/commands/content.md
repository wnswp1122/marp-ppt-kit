# 내용만 수정 (디자인 동결): $ARGUMENTS

**내용 구역의 텍스트(말)만** 수정한다. 디자인은 한 글자도 건드리지 않는다.

## 동결 규칙 (반드시 준수)
- ✅ 수정 허용: 제목·불릿·문단·코드의 **글자(말)**, 그리고 **강조 마크 `**굵게**`·`*초록*`·`==형광펜==`**(무엇을 강조할지는 내용 작성자 결정).
- 🚫 절대 금지: `<!-- _class: ... -->` Marp 지시, `<div class="...">` HTML 태그, **클래스 마커 `{.box .accent}`·`::: ... :::`**, `themes/*.css`.
- **경계 = 구문**: 클래스 마커(`{...}`·`:::`)·HTML 래퍼는 **그 자리에 그대로 두고 안의 글자만** 바꾼다. 예: `치킨이 최고다 {.box .accent}` → 글자만 고쳐 `피자가 최고다 {.box .accent}` (끝의 `{...}` 토큰은 손대지 않음 → 박스 유지).
- 단어에 딱 붙은 인라인 attrs(`치킨{.accent}`)는 워크플로우에서 지양하지만, 있으면 `{.accent}`를 그 단어에 딸린 디자인 토큰으로 보고 **그대로 데려간다**(떼거나 다른 단어로 옮기지 않음).

## 진행
1. 대상: deck-name + (번호 주면 그 파일 / 없으면 전체 슬라이드). `decks/<deck-name>/slides/`.
2. 지시사항대로 텍스트만 수정. 디자인 구역은 보존.
3. `node build.mjs <deck-name>` 빌드 확인 후, **무엇을 바꿨는지(디자인은 안 건드렸음)** 보고.

> 레이아웃·색·클래스를 바꾸려면 `/design`을 쓴다.
