# 발표 대본 작성: $ARGUMENTS

덱의 슬라이드를 읽고 **발표할 때 말할 대본**을 `decks/<deck-name>/notes.md`에 쓴다.
대본은 빌드가 각 슬라이드의 **발표자 노트**로 주입한다(발표 중 `P`키 / PPTX 발표자 노트).

## 동결 규칙 (반드시 준수)
- ✅ 수정 허용: `decks/<deck-name>/notes.md` (대본 파일) **하나만**
- 🚫 절대 금지: `slides/*.md`(내용·디자인), `themes/*.css`, `_header.md` — 슬라이드는 한 글자도 안 건드린다.
- 대본은 "제4구역"이다. 내용(말=슬라이드 텍스트)·디자인과 분리된, 발표자가 *입으로 할 말*.

## notes.md 형식 (빌드 계약)
- 장별 `## NN-이름` 헤더로 구분. **앞자리 숫자(NN)로 슬라이드와 매칭**된다 (`01-cover.md` ↔ `## 01-cover` 또는 `## 1`).
- 헤더 아래 본문이 그 슬라이드의 대본. `templates/notes.md` 참고.
- **마크다운 스타일이 발표자뷰에 그대로 렌더된다** — `**굵게**`·`- 불릿`·`> 인용`·`==형광펜==`·`` `코드` `` 사용 가능(빌드가 P키 노트로 렌더 주입).
- 빌드는 `<!-- -->` 주석으로 슬라이드 앞에 주입하므로 대본 안에 `-->`는 피한다(빌드가 자동 치환은 하지만 가독성 위해).

## 진행
1. 대상 결정: deck-name + (번호 주면 그 장만 / 없으면 전 슬라이드). 기존 `notes.md` 있으면 읽어 해당 장만 갱신.
2. 각 슬라이드 본문(제목·불릿·핵심 메시지)을 읽고, **그 장에서 말할 것을 "요약"** 한다 — 줄글 대본이 아니라 **발표자가 흘끗 보고 떠올릴 키포인트**.
   - 형식: 짧은 **불릿** 위주 + 핵심 단어 `**굵게**`. 통문장으로 읽을 대본 ❌, 한눈에 들어오는 요약 ✅.
   - 슬라이드 글자를 그대로 옮기지 말 것. 슬라이드엔 없는 "말로 보탤 것"(맥락·전환·강조·청중 질문)을 적는다.
   - 한 장당 3~5불릿 이내. 분량은 발표 시간 참고(`brief.md`, 1장당 40~60초).
   - 연기 지시(`> 한 박자 쉬고`)나 전환 멘트는 인용(`>`)으로 구분하면 보기 좋다.
3. `decks/<deck-name>/notes.md` 작성/갱신.
4. `node build.mjs <deck-name>` 빌드 → 에러 없는지 + 발표자 노트 주입됐는지 확인 후 보고.
   - 예상 발표 시간은 `pptime <deck-name>`으로 점검 권유.

> 슬라이드 내용을 바꾸려면 `/content`, 디자인은 `/design`. 이 스킬은 대본 전용.
