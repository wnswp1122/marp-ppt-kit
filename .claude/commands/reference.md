# 레퍼런스 추출: $ARGUMENTS

좋은 참고자료를 분석해 **우리 디자인 시스템(`themes/`·`themes/base.css`·`patterns/PATTERNS.md`)에 흡수**한다.
**철칙 2가지:** ① 자동 저장 금지 — **제안→승인→반영**. ② 기존 컴포넌트 **재사용 우선**, 진짜 새로운 모양일 때만 추가(`PATTERNS.md` 승격 기준).
색은 절대 직접 박지 않고 `var(--*)` 토큰으로(팔레트는 `themes/<이름>.css`).

## 0. 발동 · 입력 확인
- 사용자가 **파일만 던졌으면** 먼저 **"reference 스킬로 분석·추출할까요?"** 묻고 예일 때 진행.
  명시적 의도("추출해"·"레퍼런스로"·"이 스타일 가져와")면 바로 진행.
- 인자/대화에 파일 경로가 없으면 요청. 지원: `pdf` · `html`/`htm` · `pptx` · 이미지(`png`/`jpg`).

## 1. 1차 분석 (포맷별 — 추출 깊이가 다르다)
> PDF/이미지는 "보고 재현"(코드 없음), HTML/PPTX는 "진짜 추출"(실제 스타일).

- **PDF / 이미지** — 렌더 후 **시각 분석 + 픽셀 색 샘플링**.
  - `pip install --break-system-packages pymupdf`(fitz) → 페이지를 PNG로 렌더(`get_pixmap(dpi=…)`), 색 히스토그램으로 팔레트 추정. 폰트·정확한 CSS는 추정만.
  - 또는 받아둔 `./chrome/`로 보고 눈 분석. 장수 많으면 몽타주(그리드) 만들어 한눈에 스캔.
- **HTML** — **실제 CSS/computed style 파싱**. `./chrome/` + `puppeteer-core`로 DOM 열어 `getComputedStyle`(색·폰트·간격·radius), 반복 구조 → 컴포넌트 후보 추출.
- **PPTX** — `unzip`으로 `ppt/theme/theme1.xml`(색 6종+강조·폰트), `ppt/slideLayouts/*`(마스터 레이아웃) 파싱.
- 산출: "이 자료엔 이런 **색·폰트·레이아웃·컴포넌트·강조·서사**가 있다" 목록(근거 = 슬라이드/요소 번호).

## 2. ❓ 추출 메뉴 질문 (핵심 — 무작정 다 가져오지 않는다)
1차 분석으로 **찾은 것을 보여준 뒤** `AskUserQuestion`(복수 선택)으로 무엇을 흡수할지 받는다:
- **색상(팔레트)** → `themes/<이름>.css` 새 테마
- **폰트/타이포 위계** → 팔레트 폰트 토큰
- **레이아웃** → `base.css`의 `section.이름`
- **컴포넌트/패턴** → `base.css` + `PATTERNS.md`
- **강조 체계**(굵게·색·형광펜·밑줄) → `base.css` 강조 규칙
- **서사 구조** → brief 템플릿 (※ 아래)
> **내용(말) 자체는 가져오지 않는다**(그 발표 고유의 것). "내용이 좋다" = **구성/스토리 흐름**으로 해석해 서사 골격만 brief 템플릿화.

## 3. 선택한 것만 깊게 추출 + 우리 시스템에 매핑
- 고른 항목만 정밀 추출.
- **매핑 판단**: 기존 것으로 되면 재사용(`.box`·`.callout`·`.cols`·`section.*`), 진짜 새로우면 추가 후보.
- 색·폰트는 토큰화. 새 팔레트는 `tech.css` 복사 → `/* @theme 새이름 */` + `:root`만 교체(빌드가 자동 등록).

## 4. 제안서 → 승인 (obo 방식)
표로 제시: **[추가 / 재사용 / 새 테마]** · 무엇 · 어디(`themes/`·`base.css`·`PATTERNS.md`) · 근거.
`PATTERNS.md` 승격 기준(2~3회 반복·내용 독립·이름 가치)을 적용해 과한 추상화 막기. 항목이 많으면 `/obo`로 하나씩 짚으며 결정받는다.

## 5. 반영 + 시각 검증
- **승인된 것만** 반영(색=`var()`, 기존 컨벤션 준수). 컴포넌트는 `PATTERNS.md`에도 카탈로그 등록.
- 데모 한 장을 `decks/showcase` 또는 `decks/example/<새테마>`에 추가(선택, 테마 검증 하니스).
- `node build.mjs <덱> png`로 렌더해 **눈으로 검증**(시각 검토 패스). 넘침·정렬·대비 확인.

> 추출·반영은 **작성 패스**다. 끝나면 `/review`로 **별도 검증 패스**를 권장(self-approve 금지).
