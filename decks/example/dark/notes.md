# example 대본 (발표자 노트 데모)

> 각 노트는 *말할 것의 요약*. 발표자뷰(P키)에서 **굵게·==형광펜==·제목·불릿·인용·코드**가 그대로 렌더된다.

## 01-cover
# 여는 멘트
- 한 박자 쉬고 **"마크다운만으로 만든 발표"** 임을 한 줄로
> 표지는 설명하지 말 것

## 02-agenda
- 오늘 ==다섯 덩어리== 를 보여준다고 예고
- "마지막에 발표자 노트(지금 이 화면)도 보여드립니다" 한마디

## 03-section
- 챕터 전환. **숨 고르기** 구간
> 청중 시선 모으고 다음으로

## 04-emphasis
# 여기가 강조 데모
- **파랑**(strong) / *초록*(em) / ==형광펜==(mark) 셋의 차이를 눈으로 짚기
- 인라인 `코드` 도 된다고 한마디
> 천천히 — 이 장이 텍스트 기능의 핵심

## 05-plugins
- 핵심 메시지: ==HTML 안 써도 클래스가 붙는다==
- `{.box .accent}` 와 `::: box` 두 방식 차이 설명
- danger 박스는 "되돌릴 수 없는 것"에만 쓴다고 강조

## 06-two
- 왼쪽 설명 ↔ 오른쪽 코드 구조라고 안내
- `<!-- _class: two -->` + `<div>` 두 개라고 한 줄

## 07-compare
- 왼쪽(기존) → 오른쪽(마크다운)으로 **시선 이동** 유도
- 한 문장 결론: "텍스트가 곧 슬라이드"

## 08-table-code
- 표로 **포맷 3종** 비교
- ==HTML만 동적== 이라는 점이 포인트

## 09-metrics
# 숫자가 차오르는 장
- 슬라이드 뜨는 순간 **카운트업** 됨 — 잠깐 기다렸다 말하기
> 애니메이션 끝난 뒤 수치 언급

## 10-blank
# 인터랙티브 데모
- ==슬라이더를 직접 드래그== 하면서 설명
- 슬라이드가 늘면 **시간·불릿·에이전트 수**가 같이 변한다고 짚기
> 청중에게 "5분이면 몇 장?" 한 번 물어보기

## 11-imgtext
- 이미지는 `assets/`에 두고 빌드가 ==base64 인라인==
- "파일 하나만 공유해도 이미지 보임" 강조

## 12-full
- 풀스크린 임팩트 — **한 문장만** 크게
- 여기서 전환효과가 `slide`로 바뀐다고 (원하면) 짚기

## 13-quote
- 마무리 직전 **핵심 한 줄** 인용
> 잠깐 침묵 → 여운

## 14-diagram
- **Mermaid** 블록을 빌드가 HTML 런타임 렌더(브라우저가 그림)
- 분기·서브그래프·DB 노드 — CSS 박스로는 어려운 그래프
> HTML 전용 — PDF/PPTX엔 안 나옴

## 15-end
- 감사 인사 + `/brainstorm` 으로 새 발표 시작 가능하다고 안내
- 질문 받기
